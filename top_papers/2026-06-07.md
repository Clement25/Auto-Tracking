# Top Papers — AI Agents & Embodied AI (2026-06-07)

## [Agent Planning Benchmark: A Diagnostic Framework for Planning Capabilities in LLM Agents](https://arxiv.org/abs/2606.04874)
**Authors:** (arXiv:2606.04874)
**TL;DR:** APB is a planning-specific diagnostic benchmark with 4,209 multimodal cases across 22 domains and five planning settings, covering holistic planning, feedback-conditioned step-wise planning, and robustness under extraneous tools, broken tools, and unsolvable tasks. Evaluated across 12 MLLMs, APB reveals systematic weaknesses in long-horizon planning, tool-noise robustness, calibrated refusal, and inference-time refinement. The work provides the community with a rigorous lens for measuring where agent planning actually breaks down.
**Tags:** `agents`, `planning`, `benchmarks`, `LLM`, `tool-use`

## [The End of Software Engineering: How AI Agents Are Restructuring the Software Paradigm](https://arxiv.org/pdf/2606.05608)
**Authors:** (arXiv:2606.05608)
**TL;DR:** This paper argues that the rise of AI agents — where large language models serve as the primary reasoning engine, dynamically generating and discarding code — constitutes a fundamental restructuring of the software development paradigm rather than a mere productivity boost. The authors examine how this shift dissolves traditional roles such as the programmer, architect, and tester, replacing them with agent orchestration and prompt engineering. The paper calls for rethinking CS education, tooling, and software methodology in light of this transformation.
**Tags:** `agents`, `software-engineering`, `LLM`, `future-of-work`

## [TempoVLA: Learning Speed-Controllable Vision-Language-Action Policies](https://arxiv.org/abs/2606.06491)
**Authors:** (arXiv:2606.06491)
**TL;DR:** TempoVLA observes that robot manipulation alternates between low-risk transit phases and high-risk contact stages, yet existing Vision-Language-Action models inherit only a single fixed execution speed from training demonstrations. The paper proposes a speed-controllable VLA policy that dynamically adjusts action cadence based on task phase, improving both safety and efficiency during contact-rich manipulation. Experiments demonstrate significant gains in task success rate on real and simulated robot benchmarks.
**Tags:** `embodied-ai`, `robotics`, `VLA`, `manipulation`, `vision-language-action`

## [FlowPRO: Reward-Free Reinforced Fine-Tuning of Flow-Matching VLAs via Proximalized Preference Optimization](https://arxiv.org/html/2606.05468)
**Authors:** (arXiv:2606.05468)
**TL;DR:** FlowPRO tackles the challenge of post-training flow-matching Vision-Language-Action models into deployable robot policies without requiring explicit reward functions. It introduces a proximalized preference optimization scheme that aligns VLA behavior with human preferences directly from demonstrations, bypassing brittle reward engineering. The method yields policies with higher real-world deployment reliability on dexterous manipulation tasks.
**Tags:** `embodied-ai`, `robotics`, `VLA`, `RLHF`, `flow-matching`, `fine-tuning`

## [Intercepting the Future: Latent-Space Predictive World Model for Dynamic VLA Manipulation](https://arxiv.org/html/2606.02486v1)
**Authors:** (arXiv:2606.02486)
**TL;DR:** Existing VLA models generalize well across static manipulation but fail catastrophically when objects move during task execution. This paper introduces a latent-space predictive world model that anticipates object trajectories and pre-computes interception waypoints, enabling dynamic in-flight re-planning. Evaluations on moving-target manipulation benchmarks show the approach substantially outperforms reactive baselines.
**Tags:** `embodied-ai`, `robotics`, `VLA`, `world-models`, `manipulation`, `dynamic-environments`

## [From Human Videos to Robot Manipulation: A Survey on Scalable Vision-Language-Action Learning with Human-Centric Data](https://arxiv.org/html/2606.00054)
**Authors:** (arXiv:2606.00054)
**TL;DR:** This survey covers the emerging paradigm of training robot manipulation policies from large-scale human video data, including contrastive latent action pretraining (CLAP) and related methods that bypass the need for robot-collected demonstrations. It systematically reviews datasets, architectures, and evaluation protocols, identifying key bottlenecks in bridging the human-to-robot embodiment gap. The paper serves as a roadmap for scaling robot learning using internet-scale video.
**Tags:** `embodied-ai`, `robotics`, `VLA`, `data-scaling`, `imitation-learning`, `human-video`

## [Extending AI for Research to the Humanities: A Multi-Agent Framework for Evidence-Grounded Scholarship](https://arxiv.org/abs/2605.30947)
**Authors:** (arXiv:2605.30947)
**TL;DR:** SPIRE addresses the challenge of applying AI research agents to humanities scholarship, which demands interpretive, evidence-grounded argument over primary sources with faithful quotation and verifiable provenance. The framework decomposes recurring humanities operations into cooperating agent roles — source discovery, evidence annotation, comparison, and provenance checking — and shows strong performance on literature and history analysis tasks. This extends agentic AI research beyond STEM domains into the broader social sciences.
**Tags:** `agents`, `multi-agent`, `humanities`, `RAG`, `provenance`

## [ToolTree: Efficient LLM Agent Tool Planning via Dual-Feedback Monte Carlo Tree Search and Bidirectional Pruning](https://arxiv.org/html/2603.12740v1)
**Authors:** (arXiv:2603.12740)
**TL;DR:** ToolTree introduces a Monte Carlo tree search-inspired planning paradigm for tool selection in LLM agents, using a dual-stage LLM evaluation and bidirectional pruning mechanism to explore possible tool usage trajectories efficiently. The approach enables agents to make informed, adaptive decisions over extended tool-use sequences without exponential search cost. Benchmarks across API-calling tasks show ToolTree achieving state-of-the-art accuracy with lower average tool calls.
**Tags:** `agents`, `tool-use`, `planning`, `MCTS`, `LLM`

## [Agentic Tool Use in Large Language Models](https://arxiv.org/html/2604.00835v1)
**Authors:** (arXiv:2604.00835)
**TL;DR:** This paper surveys how LLMs are augmented with planning, memory, and action capabilities — with tool use as the key operationalizing mechanism for invoking external APIs, databases, and code interpreters. It introduces a taxonomy of agentic tool-use patterns from single-call to multi-turn orchestration, and analyzes failure modes including hallucinated tool calls, cascading errors, and context window saturation. The survey contextualizes recent systems like GPT-4o, Claude, and Gemini within this framework.
**Tags:** `agents`, `tool-use`, `LLM`, `survey`, `planning`

## [The Evolution of Tool Use in LLM Agents: From Single-Tool Call to Multi-Tool Orchestration](https://arxiv.org/html/2603.22862v1)
**Authors:** (arXiv:2603.22862)
**TL;DR:** This paper traces how evaluation paradigms for multi-tool agents have undergone a decisive shift — from measuring isolated API invocation correctness to assessing system-level intelligence in sustaining, adapting, and repairing extended tool-use trajectories. The authors introduce a new benchmark suite testing recovery from mid-trajectory tool failures and context drift over long agentic sessions. Results reveal that current frontier models still struggle with graceful degradation when tool chains partially fail.
**Tags:** `agents`, `tool-use`, `multi-agent`, `benchmarks`, `LLM`, `orchestration`
