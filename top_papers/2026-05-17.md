# Top Papers — AI Agents & Embodied AI (2026-05-17)

## [World Action Models: The Next Frontier in Embodied AI](https://arxiv.org/abs/2605.12090)
**Authors:** (arXiv:2605.12090)
**TL;DR:** Proposes World Action Models (WAMs), embodied foundation models that unify predictive state modeling with action generation by targeting a joint distribution over future states and actions rather than actions alone. This approach moves beyond current VLA models that generate only actions, enabling richer reasoning about physical consequences before committing to a behavior. WAMs represent the next architectural leap for grounding AI decision-making in the physical world.
**Tags:** `embodied-ai`, `world-models`, `VLA`, `robotics`, `foundation-models`

## [MIND-Skill: Quality-Guaranteed Skill Generation via Multi-Agent Induction and Deduction](https://arxiv.org/abs/2605.08670)
**Authors:** (arXiv:2605.08670)
**TL;DR:** Introduces MIND-Skill, a framework that automatically induces generalizable skills from successful agent trajectories using a multi-agent induction-deduction loop with built-in quality guarantees. The system extracts reusable behavior primitives without manual annotation, enabling agents to accumulate a growing skill library over time. This addresses a core bottleneck in long-horizon autonomous task completion.
**Tags:** `agents`, `skill-learning`, `multi-agent`, `LLM`, `planning`

---

## [BioProVLA-Agent: Affordable VLA-Enabled Embodied Multi-Agent for Biological Lab Manipulation](https://arxiv.org/abs/2605.07306)
**Authors:** (arXiv:2605.07306)
**TL;DR:** Presents BioProVLA-Agent, a low-cost (~$850 hardware) protocol-driven embodied multi-agent system that chains a Guiding Decision Agent, a Protocol Agent, a VLM-RAG Verification Agent, and a VLA Embodied Agent to autonomously execute wet-lab procedures. The paper introduces AugSmolVLA, an online augmentation strategy designed for transparent labware and challenging lab lighting conditions. Evaluated on 15 atomic tasks and 6 composite workflows, it outperforms ACT, X-VLA, and SmolVLA baselines.
**Tags:** `embodied-ai`, `robotics`, `VLA`, `multi-agent`, `science-automation`

## [ExploitGym: Can AI Agents Turn Security Vulnerabilities into Real Attacks?](https://arxiv.org/abs/2605.11086)
**Authors:** (arXiv:2605.11086)
**TL;DR:** ExploitGym is a large-scale benchmark that evaluates AI agents' ability to extend vulnerability triggers into working exploits, covering a diverse range of real-world CVEs. The benchmark reveals significant gaps between state-of-the-art agent capabilities and the level of reasoning required for reliable exploit development. Results provide a grounded baseline for tracking autonomous cybersecurity AI capabilities.
**Tags:** `agents`, `security`, `benchmarks`, `exploit`, `LLM`

## [Cattle Trade: A Multi-Agent Benchmark for LLM Bluffing, Bidding, and Bargaining](https://arxiv.org/abs/2605.14537)
**Authors:** (arXiv:2605.14537)
**TL;DR:** Introduces a long-horizon (50–60 turn) multi-agent economic game that simultaneously tests LLM agents on auctions, hidden-offer trade, bargaining, bluffing, opponent modeling, and resource allocation under imperfect information. Unlike prior benchmarks that evaluate these skills in isolation, Cattle Trade forces agents to integrate them within a competitive game with conflicting incentives. This provides a much more realistic test of strategic reasoning in autonomous agents.
**Tags:** `agents`, `benchmarks`, `multi-agent`, `strategic-reasoning`, `LLM`

## [GroupMemBench: Benchmarking LLM Agent Memory in Multi-Party Conversations](https://arxiv.org/abs/2605.14498)
**Authors:** (arXiv:2605.14498)
**TL;DR:** Addresses the gap between single-user memory benchmarks and real-world multi-user deployments by introducing GroupMemBench, which tests LLM agent memory in group chat and collaborative workspace scenarios. Key challenges include speaker-grounded belief tracking and audience-adapted response generation across dozens of participants. This benchmark is critical as agents increasingly serve as workplace collaborators rather than individual assistants.
**Tags:** `agents`, `memory`, `benchmarks`, `multi-party`, `LLM`

## [Portable Agent Memory: Provenance-Verified Memory Transfer Across Heterogeneous LLM Agents](https://arxiv.org/abs/2605.11032)
**Authors:** (arXiv:2605.11032)
**TL;DR:** Proposes an open protocol for serializing, transporting, and re-hydrating agent memory across heterogeneous LLM runtimes with cryptographic integrity guarantees. The protocol defines a five-component memory model (episodic, semantic, procedural, working, identity) that allows valuable operational context to travel with the agent between vendors and frameworks. This is foundational infrastructure for persistent, cross-platform autonomous agents.
**Tags:** `agents`, `memory`, `protocols`, `LLM`, `interoperability`

## [Beyond the Black Box: Interpretability of Agentic AI Tool Use](https://arxiv.org/abs/2605.06890)
**Authors:** (arXiv:2605.06890)
**TL;DR:** Develops a mechanistic interpretability toolkit using Sparse Autoencoders (SAEs) and linear probes to predict whether an agent will invoke a tool and how consequential that tool call will be—before the action is taken. The approach addresses common failure modes where agents skip required tool calls, invoke them unnecessarily, or take high-stakes irreversible actions. This provides an early-warning layer for safer agentic deployment.
**Tags:** `agents`, `interpretability`, `tool-use`, `safety`, `mechanistic-interpretability`

## [SAGA: Workflow-Atomic Scheduling for AI Agent Inference on GPU Clusters](https://arxiv.org/abs/2605.00528)
**Authors:** (arXiv:2605.00528)
**TL;DR:** Proposes shifting GPU scheduling granularity from individual LLM requests to entire agent workflows, treating the multi-step reasoning chain as the first-class schedulable unit. SAGA achieves significant throughput improvements on agent workloads that chain tens to hundreds of LLM calls per task. This systems-level contribution is critical for cost-effective large-scale agentic deployment.
**Tags:** `agents`, `systems`, `inference`, `scheduling`, `LLM`

## [FP-Agent: Fingerprinting AI Browsing Agents](https://arxiv.org/abs/2605.01247)
**Authors:** (arXiv:2605.01247)
**TL;DR:** Conducts a controlled measurement study of seven AI browsing agents performing real tasks (flight booking, shopping, forum interaction) to characterize how they differ from human browser fingerprints. The paper identifies reliable behavioral and browser signals that distinguish agent traffic from humans, with implications for both security and access policy design. As web agents proliferate, understanding their detectability is increasingly important.
**Tags:** `agents`, `web-agents`, `security`, `fingerprinting`, `browser`

## [Autonomy and Agency in Agentic AI: Architectural Tactics for Regulated Contexts](https://arxiv.org/abs/2605.12105)
**Authors:** (arXiv:2605.12105)
**TL;DR:** Characterizes agentic AI systems by their multi-step planning, external tool invocation, persistent memory, and cross-environment coordination, then derives architectural patterns for deploying them safely in regulated industries (finance, healthcare, defense). The paper maps autonomy levels to compliance requirements and proposes concrete design tactics such as audit trails, sandboxed tool execution, and human-in-the-loop escalation points. A practical guide for enterprise-grade agentic system design.
**Tags:** `agents`, `architecture`, `safety`, `regulation`, `enterprise`

## [Security Risks in Tool-Enabled AI Agents: A Systematic Analysis of Privileged Execution Environments](https://arxiv.org/abs/2605.09721)
**Authors:** (arXiv:2605.09721, accepted at IEEE COMPSAC 2026)
**TL;DR:** Provides a systematic taxonomy of security risks arising when AI agents are granted privileged execution in cloud environments, covering risks from prompt injection to tool-chaining attacks and data exfiltration. The analysis includes a threat model for cloud-hosted agents and a set of mitigation strategies covering sandboxing, capability restriction, and anomaly detection. Essential reading for teams deploying production AI agents with real-world tool access.
**Tags:** `agents`, `security`, `cloud`, `tool-use`, `threat-modeling`
