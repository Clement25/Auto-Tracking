# Top Papers — AI Agents & Embodied AI (2026-06-18)

## [Memory for Autonomous LLM Agents: Mechanisms, Evaluation, and Emerging Frontiers](https://arxiv.org/abs/2603.07670)
**Authors:** (Survey — multiple contributors, arXiv 2603.07670)
**TL;DR:** A comprehensive survey formalizing agent memory as a write-manage-read loop, examining five mechanism families: context-resident compression, retrieval-augmented stores, reflective self-improvement, hierarchical virtual context, and policy-learned management. Covers work from 2022 through early 2026 and maps open evaluation frontiers for long-horizon autonomous agents.
**Tags:** `agents`, `llm`, `memory`, `survey`, `long-horizon`

## [Multi-Layered Memory Architectures for LLM Agents: An Experimental Evaluation of Long-Term Context Retention](https://arxiv.org/abs/2603.29194)
**Authors:** (arXiv 2603.29194)
**TL;DR:** Presents a Multi-Layer Memory Framework that decomposes dialogue history into working, episodic, and semantic layers with adaptive retrieval gating and retention regularization. Experimental evaluations show significant gains in long-term context retention compared to flat memory baselines.
**Tags:** `agents`, `memory`, `llm`, `context-retention`, `multi-layer`

---

## [Governing Evolving Memory in LLM Agents: Risks, Mechanisms, and the SSGM Framework](https://arxiv.org/abs/2603.11768)
**Authors:** (arXiv 2603.11768)
**TL;DR:** Identifies critical safety and scalability risks in evolving memory systems for LLM agents, including memory control-flow attacks where retrieved memories can override explicit user instructions. Proposes the Stability and Safety Governed Memory (SSGM) framework to enforce invariants on what agents can store, retrieve, and act upon.
**Tags:** `agents`, `memory`, `safety`, `governance`, `llm`

---

## [A Survey on the Security of Long-Term Memory in LLM Agents: Toward Mnemonic Sovereignty](https://arxiv.org/abs/2604.16548)
**Authors:** (arXiv 2604.16548)
**TL;DR:** Surveys emerging attack vectors against persistent agent memory, including prompt-injection via memory retrieval and adversarial memory planting. Introduces the concept of "mnemonic sovereignty" — the principle that users should retain final authority over what an agent remembers and acts upon.
**Tags:** `agents`, `security`, `memory`, `llm`, `adversarial`

---

## [Runtime Governance for AI Agents: Policies on Paths](https://arxiv.org/abs/2603.16586)
**Authors:** (arXiv 2603.16586)
**TL;DR:** Proposes a formal runtime governance framework that monitors LLM agent execution traces and enforces policy constraints in real time, analogous to OS-level process management. The system can interrupt, redirect, or terminate agent action sequences that violate safety or authorization policies without retraining the underlying model.
**Tags:** `agents`, `governance`, `safety`, `runtime`, `policy`

## [Workspace-Bench 1.0: Benchmarking AI Agents on Workspace Tasks with Large-Scale File Dependencies](https://arxiv.org/abs/2605.03596)
**Authors:** (arXiv 2605.03596)
**TL;DR:** Introduces Workspace-Bench, a realistic benchmark evaluating AI agents on office-like tasks involving large file systems with complex interdependencies (spreadsheets, code repos, email threads). Reveals that current LLM agents struggle significantly with dependency resolution and multi-step file manipulation beyond toy settings.
**Tags:** `agents`, `benchmark`, `evaluation`, `tool-use`, `workspace`

## [MarketBench: Evaluating AI Agents as Market Participants](https://arxiv.org/abs/2604.23897)
**Authors:** (arXiv 2604.23897)
**TL;DR:** Proposes MarketBench, a simulation environment where LLM-based agents act as buyers, sellers, and market makers in synthetic economic markets. Evaluates agents on strategic reasoning, pricing, and multi-round negotiation, finding significant gaps between current models and human-level market participation.
**Tags:** `agents`, `benchmark`, `economics`, `multi-agent`, `reasoning`

## [CORAL: Self-Evolving Multi-Agent Systems via Persistent Shared Memory](https://arxiv.org/list/cs.MA/recent)
**Authors:** (arXiv — recent cs.MA submission)
**TL;DR:** CORAL introduces long-running multi-agent systems that self-evolve through shared persistent memory, asynchronous execution, and heartbeat-based interventions among agents. Achieves 3–10× higher improvement rates compared to fixed evolutionary-search baselines in complex task domains.
**Tags:** `multi-agent`, `self-evolving`, `memory`, `agents`, `llm`

## [DyTopo: Dynamic Agent Communication Topology via Semantic Matching](https://arxiv.org/list/cs.MA/recent)
**Authors:** (arXiv — recent cs.MA submission)
**TL;DR:** DyTopo investigates dynamically rewiring agent-to-agent connections at each reasoning round using semantic matching, replacing fixed communication topologies in multi-agent LLM systems. Experiments show improved task performance and reduced communication overhead across diverse collaborative reasoning benchmarks.
**Tags:** `multi-agent`, `communication`, `topology`, `llm`, `reasoning`

## [Vision-Language-Action in Robotics: A Survey of Datasets, Models, and Challenges](https://arxiv.org/abs/2604.23001)
**Authors:** (arXiv 2604.23001)
**TL;DR:** Comprehensive 2026 survey of Vision-Language-Action (VLA) models for robotic manipulation, cataloging datasets, architectures, and evaluation protocols across a rapidly expanding literature. Identifies that the core bottleneck has shifted from model design to data quality and standardized evaluation, with existing datasets varying widely in modality coverage, action representations, and robot platforms.
**Tags:** `embodied-ai`, `robotics`, `vision-language-action`, `survey`, `manipulation`

## [AutoNumerics: A Multi-Agent Pipeline for End-to-End Numerical Solver Synthesis](https://arxiv.org/list/cs.MA/recent)
**Authors:** (arXiv — recent submission)
**TL;DR:** AutoNumerics is a multi-agent system that reads PDE problem descriptions in plain text, then writes, debugs, and validates classical numerical solvers end-to-end without human intervention. The pipeline combines a planner agent, a coder agent, and a verifier agent, demonstrating strong performance on a diverse suite of PDE benchmarks.
**Tags:** `agents`, `tool-use`, `scientific-computing`, `code-generation`, `multi-agent`
