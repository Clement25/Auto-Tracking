# Top Papers — AI Agents & Embodied AI (2026-05-19)

## [AI Agents May Always Fall for Prompt Injections](https://arxiv.org/abs/2605.17634)
**Authors:** Sahar Abdelnabi, Eugene Bagdasarian
**TL;DR:** This paper recasts prompt injection attacks through the lens of Contextual Integrity (CI) theory, showing that the prevailing defense paradigm of data-instruction separation fundamentally fails against contextual manipulation. The authors prove an impossibility result: an adversary can always construct a context where a blocked flow appears legitimate, making prompt injection a structurally unsolvable problem for deployed agents. The work categorizes novel attack scenarios—misrepresenting flows, manipulating norms, and mixing flows—that existing defenses cannot detect.
**Tags:** `agents`, `security`, `prompt-injection`, `LLM`, `adversarial`

## [World Action Models: The Next Frontier in Embodied AI](https://arxiv.org/abs/2605.12090)
**Authors:** Siyin Wang, Junhao Shi, Zhaoyang Fu, Xinzhe He, Feihong Liu, Chenchen Yang, Yikang Zhou, Zhaoye Fei, Jingjing Gong, Jinlan Fu, Mike Zheng Shou, Xuanjing Huang, Xipeng Qiu, Yu-Gang Jiang
**TL;DR:** This survey introduces World Action Models (WAMs), embodied foundation models that unify predictive state modeling with action generation by targeting a joint distribution over future states and actions—rather than actions alone. Unlike standard VLA models that learn reactive observation-to-action mappings, WAMs explicitly model how the physical world evolves under intervention. The paper provides a comprehensive taxonomy of current approaches and identifies key open challenges in grounding, data efficiency, and real-world transfer.
**Tags:** `embodied-ai`, `world-models`, `robotics`, `VLA`, `survey`

## [From Storage to Experience: A Survey on the Evolution of LLM Agent Memory Mechanisms](https://arxiv.org/abs/2605.06716)
**Authors:** (Multiple authors, May 2026)
**TL;DR:** This survey proposes an evolutionary three-stage framework for LLM agent memory: Storage (raw trajectory preservation), Reflection (trajectory refinement through self-critique), and Experience (high-level abstraction enabling generalization). The framework unifies a wide body of existing work and highlights how memory architecture is the critical bottleneck for long-horizon agentic tasks. The authors identify key open problems in memory consolidation, retrieval efficiency, and cross-task transfer.
**Tags:** `agents`, `memory`, `LLM`, `survey`, `long-horizon`

## [Agentic AI and the Industrialization of Cyber Offense](https://arxiv.org/abs/2605.06713)
**Authors:** (Multiple authors, May 2026)
**TL;DR:** This paper examines how agentic AI systems—capable of planning, tool use, code inspection, and multi-step coordination—are fundamentally transforming the cybersecurity threat landscape by compressing the attack lifecycle. Reconnaissance, phishing, credential abuse, vulnerability triage, and exploit adaptation can now be orchestrated at dramatically lower cost and higher scale. The paper proposes a prioritized defensive framework for enterprises and mid-market companies (Mittelstand) facing this new asymmetric threat.
**Tags:** `agents`, `cybersecurity`, `agentic-AI`, `offense`, `enterprise`

## [MARLIN: Multi-Agent Game-Theoretic Reinforcement Learning for Sustainable LLM Inference](https://arxiv.org/abs/2605.13496)
**Authors:** (Multiple authors, May 2026)
**TL;DR:** MARLIN co-optimizes time-to-first-token (TTFT), carbon emissions, water usage, and energy costs for LLM inference in cloud datacenters using a multi-agent game-theoretic reinforcement learning framework. Experiments demonstrate at least 18% reduction in TTFT and 33% reduction in carbon emissions compared to baselines. This is one of the first works to jointly address performance and sustainability objectives in large-scale LLM serving.
**Tags:** `multi-agent`, `LLM`, `inference`, `sustainability`, `reinforcement-learning`

## [Making OpenAPI Documentation Agent-Ready: Detecting Documentation and REST Smells with a Multi-Agent LLM System](https://arxiv.org/abs/2605.14312)
**Authors:** (Multiple authors, May 2026)
**TL;DR:** Motivated by the rapid adoption of the Model Context Protocol (MCP) to expose REST APIs as agent-consumable tools, this paper identifies systematic quality problems ("REST smells") in OpenAPI documentation that prevent AI agents from reliably calling APIs. A multi-agent LLM pipeline is proposed to detect and remediate these smells automatically. Evaluation on real-world APIs shows significant improvements in agent task completion rates after documentation repair.
**Tags:** `agents`, `MCP`, `API`, `REST`, `software-engineering`

## [Do Agents Need to Plan Step-by-Step? Rethinking Planning Horizon in Data-Centric Tool Calling](https://arxiv.org/abs/2605.08477)
**Authors:** (Multiple authors, May 2026)
**TL;DR:** This paper challenges the assumption that LLM agents always benefit from fine-grained step-by-step planning when solving data-centric tasks via tool calls. The authors show that mismatch between planning granularity and data source logic leads to systematic errors, and that adaptive planning horizons—matched to the vocabulary constraints of each tool—yield substantially better outcomes. The findings have practical implications for agent system design in analytics and data science workflows.
**Tags:** `agents`, `planning`, `tool-use`, `LLM`, `data-centric`

## [BioProVLA-Agent: Affordable Embodied Multi-Agent System for Biological Laboratory Manipulation](https://arxiv.org/abs/2605.07306)
**Authors:** (Multiple authors, May 8, 2026)
**TL;DR:** BioProVLA-Agent is an embodied multi-agent system powered by Vision-Language-Action (VLA) models, designed specifically for automating biological laboratory protocols. The system uses lab protocols as the task interface and integrates protocol parsing, visual state verification, and closed-loop embodied execution. The approach significantly reduces the hardware and expertise barrier for robotic automation in life sciences settings.
**Tags:** `embodied-ai`, `robotics`, `VLA`, `biology`, `multi-agent`, `manipulation`

## [Cattle Trade: A Multi-Agent Benchmark for LLM Bluffing, Bidding, and Bargaining](https://arxiv.org/abs/2605.14537)
**Authors:** (Multiple authors, May 2026)
**TL;DR:** Cattle Trade introduces a multi-agent benchmark designed to stress-test LLM agents in environments with conflicting incentives, uncertainty, and economic dynamics—specifically bluffing, bidding, and negotiation scenarios. The benchmark reveals significant weaknesses in current frontier models under adversarial economic conditions, where agents frequently fail to maintain consistent strategies or model opponent intent effectively. This work fills a gap in agent evaluation beyond cooperative task completion.
**Tags:** `agents`, `benchmark`, `multi-agent`, `LLM`, `game-theory`, `evaluation`

## [Predictive Maps of Multi-Agent Reasoning: A Successor-Representation Spectrum for LLM Communication Topologies](https://arxiv.org/abs/2605.11453)
**Authors:** (Multiple authors, May 2026)
**TL;DR:** This paper introduces a successor-representation framework for analyzing and predicting the performance of different multi-agent LLM communication topologies—chain, star, mesh, and others. The framework provides actionable diagnostics for choosing communication structures based on task type and agent capability. Empirical results show that topology choice can account for more performance variance than model size in multi-agent settings.
**Tags:** `multi-agent`, `LLM`, `communication`, `reasoning`, `architecture`

## [PostTrainBench: Can LLM Agents Automate LLM Post-Training?](https://arxiv.org/abs/2603.08640)
**Authors:** (Multiple authors, March 2026)
**TL;DR:** PostTrainBench evaluates whether autonomous LLM agents—capable of reasoning, writing code, operating developer tools, and executing multi-hour workflows—can reliably automate the post-training pipeline (fine-tuning, RLHF, evaluation). Results show current agents handle narrow post-training sub-tasks well but fail at full-pipeline orchestration requiring adaptive decision-making. The benchmark establishes a new frontier task for agentic AI systems in the ML infrastructure domain.
**Tags:** `agents`, `LLM`, `post-training`, `benchmark`, `automation`
