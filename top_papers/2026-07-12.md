# Top Papers — AI Agents & Embodied AI (2026-07-12)

## [RynnWorld-4D: 4D Embodied World Models for Robotic Manipulation](https://arxiv.org/abs/2607.06559)
**Authors:** Deli Zhao, Xin Li, Siteng Huang, Xingyue Zhao, Haoyu Zhao (Alibaba DAMO Academy)
**TL;DR:** Introduces a projective 4D representation for embodied world models that synchronously predicts RGB, depth, and optical flow, trained on a newly curated dataset of 254.4 million frames from egocentric human and robotic manipulation videos. The RynnWorld-4D-Policy model consumes internal 4D representations in a single forward pass — bypassing expensive multi-step denoising — to output robot actions in closed-loop. Achieves state-of-the-art performance on real-world dexterous bimanual manipulation tasks.
**Tags:** `embodied-ai`, `world-models`, `robotics`, `manipulation`, `4D-representation`

## [Multi-Agent Robotic Control with Onboard Vision-Language Models](https://arxiv.org/abs/2607.07403)
**Authors:** Kajetan Rachwał, Maciej Majek, Bartłomiej Boczek, Jakub Matejczyk, Dominik Matejkowski, Adam Dąbrowski, Tim Seyde, Alexander Amini, Maria Ganzha
**TL;DR:** Presents a Multi-Agent System (MAS) architecture for robotic control that deploys specialized VLM and VLA agents entirely on onboard hardware, eliminating dependence on external compute. A novel "Megamind" orchestration agent mitigates context retention issues for long-horizon planning using compact VLMs (3–20B parameters), tested across five task categories in a simulated industrial warehouse. The system covers safety inspection, warehouse maintenance, package quality verification, and responding to human requests.
**Tags:** `agents`, `embodied-ai`, `robotics`, `VLM`, `multi-agent`, `onboard-inference`

## [ASPIRE: Agentic Skills Discovery for Robotics](https://arxiv.org/abs/2607.00272)
**Authors:** Runyu Lu, Yubo Wu, Ethan Kou, Letian Fu, Wenli Xiao, Ajay Mandlekar, Yinzhen Xu, Guanya Shi, Ken Goldberg, Ang Chen, Mosharaf Chowdhury, Yuke Zhu, Linxi "Jim" Fan, Guanzhi Wang (NVIDIA, UC Berkeley, UIUC, CMU, UMich)
**TL;DR:** A continual learning system that autonomously writes and refines robot control programs in a code-as-policy paradigm, compounding experience into a reusable skill library. Three components work in concert: a closed-loop execution engine with multimodal failure diagnostics, an expanding skill library that distills validated fixes into transferable knowledge, and evolutionary search that generates diverse task sequences to explore beyond single-trajectory refinement. Bridges AI agents and embodied robotics in a single open-ended system.
**Tags:** `agents`, `embodied-ai`, `robotics`, `continual-learning`, `code-as-policy`, `skill-discovery`

## [Embodied.cpp: A Portable Inference Runtime of Embodied AI Models on Heterogeneous Robots](https://arxiv.org/abs/2607.02501)
**Authors:** Ling Xu, Chuyu Han, Borui Li, Hao Wu, Shiqi Jiang, Ting Cao, Chuanyou Li, Sheng Zhong, Shuai Wang (Southeast University, Nanjing University, Microsoft Research, Tsinghua AIIR)
**TL;DR:** Practical deployment of VLA and world-action models (WAMs) remains fragmented across model-specific Python stacks, backend assumptions, and robot-side glue code — especially on heterogeneous edge devices. Embodied.cpp is a portable C++ inference runtime organized into five layers (input adapters, sequence builders, backbone execution, head plugins, deployment adapters) that addresses multi-rate execution in closed-loop control and achieves latency-first batch-1 inference on heterogeneous hardware. Directly tackles the "last mile" of embodied AI deployment.
**Tags:** `embodied-ai`, `inference-runtime`, `robotics`, `edge-deployment`, `VLA`

## [SPEAR: A Simulator for Photorealistic Embodied AI Research](https://arxiv.org/abs/2607.06701)
**Authors:** Mike Roberts, Renhan Wang, Rushikesh Zawar, et al. (Adobe Research, Intel Labs, NVIDIA, ETH Zurich, Imperial College London)
**TL;DR:** Existing photorealistic simulators suffer from limited generality, programmability, and rendering speed. SPEAR is a Python library that connects to and programmatically controls any Unreal Engine application via a modular plugin architecture, enabling interactive simulation for training embodied agents and generating large-scale synthetic visual data. Accepted at ECCV 2026.
**Tags:** `embodied-ai`, `simulation`, `photorealistic`, `robotics`, `synthetic-data`

## [Safety Testing LLM Agents at Scale: From Risk Discovery to Evidence-Grounded Verification](https://arxiv.org/abs/2607.01793)
**Authors:** Yunhao Feng, Ruixiao Lin, Ming Wen, Qinqin He, Yanming Guo, Yifan Ding, Yutao Wu, Jialuo Chen, Zhuoer Xu, Xiaohu Du, Jianan Ma, Zixing Chen, Xingjun Ma, Yunhao Chen, Xinhao Deng
**TL;DR:** Presents Vera, an end-to-end automated safety testing framework for LLM agents that go beyond expert-designed rules through a three-stage, self-reinforcing pipeline covering risk discovery, test generation, and evidence-grounded verification. As LLM agents increasingly perform autonomous actions through external tools, existing rule-based safety tests are too costly to extend as agents evolve. Vera instantiates software engineering testing principles adapted for the non-deterministic, tool-augmented agent setting.
**Tags:** `agents`, `safety`, `evaluation`, `testing`, `LLM`

## [Atomic Task Graph: A Unified Framework for Agentic Planning and Execution](https://arxiv.org/abs/2607.01942)
**Authors:** Yue Zhang, Sihan Chen, Ziwen Huang, Hanyun Cui, Kangye Ji, Zhi Wang
**TL;DR:** Proposes the Atomic Task Graph (ATG), a unified control framework that maintains an explicit dependency graph enabling task decomposition via recursive DAGs, verified intermediate result reuse without recomputation, and principled rollback when subtasks fail. Unlike approaches that rely on scaling larger models or task-specific fine-tuning, ATG operates as a structured execution layer over any LLM backbone. Targets long-horizon, multi-step tasks where implicit planning fails silently.
**Tags:** `agents`, `planning`, `task-graphs`, `execution`, `LLM`

## [What Do AI Agents Actually Change? An Empirical Taxonomy of Mutation Patterns in Performance-Improving Pull Requests](https://arxiv.org/abs/2607.05666)
**Authors:** Illia Dovhoshliubnyi, Nima Soroush, Ashkan Sami, Alexander Brownlee
**TL;DR:** Analyzes 1,254 performance-relevant diff hunks from 216 performance-improving pull requests across five AI agent systems to characterize what coding agents actually modify in production codebases. Three mutation categories dominate: name modification (37.0%), object creation (26.4%), and type change (22.7%). Provides an empirical taxonomy reusable for evaluating and benchmarking AI coding agent behavior at scale.
**Tags:** `agents`, `coding-agents`, `empirical-analysis`, `software-engineering`

## [Agent Planning Benchmark: A Diagnostic Framework for Planning Capabilities in LLM Agents](https://arxiv.org/abs/2606.04874)
**Authors:** Haoyu Sun, Wenxuan Wang, Mingyang Song, Jujie He, Weinan Zhang, Yang Liu, Yang Yang, Yu Cheng (Tongji University, Shanghai AI Lab, HIT, Fudan, Skywork AI, UCSC, SJTU)
**TL;DR:** Introduces APB, a planning-specific diagnostic benchmark with 4,209 multimodal cases across 22 domains and five settings — holistic planning, feedback-conditioned step-wise planning, and robustness under extraneous tools, broken tools, and unsolvable tasks. Evaluated across 12 MLLMs, APB exposes systematic weaknesses in long-horizon planning, tool-noise robustness, calibrated refusal, and inference-time refinement. APB-guided refinement consistently improves plan correctness and downstream execution metrics across three representative models.
**Tags:** `agents`, `benchmark`, `planning`, `evaluation`, `multimodal`

## [Harnessing the Collective Intelligence of AI Agents in the Wild for New Discoveries](https://arxiv.org/abs/2606.10402)
**Authors:** Federico Bianchi, Yongchan Kwon, Aneesh Pappu, James Zou
**TL;DR:** Introduces EinsteinArena, an agent-native open platform for distributed research where multiple AI agents collaboratively tackle open scientific problems through shared leaderboards and discussion forums — mirroring how human researchers share partial results and build on each other's insights. As of May 2026, EinsteinArena agents have discovered 12 new state-of-the-art results exceeding any prior human or AI solution. Demonstrates that collective agent organization, not just individual capability, is key to scientific breakthrough.
**Tags:** `agents`, `collective-intelligence`, `scientific-discovery`, `multi-agent`, `open-platform`

## [Agon: An Autonomous Large-Scale Omnidisciplinary Research System Built on Prompt Economy](https://arxiv.org/abs/2606.24177)
**Authors:** Youran Sun, Xingyu Ren, Chugang Yi, Jiaxuan Guo, Kejia Zhang, Jianda Du, Haizhao Yang (University of Maryland, Chinese University of Hong Kong, Stanford University)
**TL;DR:** Presents Agon, a research orchestrator built on six design principles — Prompt Economy, Future-Facing, Minimal Prompts, OmniDisciplinary, Massive Parallelism, and Zero-Code — that deploys large-scale autonomous agent workflows across scientific disciplines without per-task human tuning. Validates results using automated verifiers inside the workflow and escalates unresolved judgments to human scientists, targeting a dramatic reduction in token costs while maintaining research quality at scale.

## [SEEA-R1: Tree-Structured Reinforcement Fine-Tuning for Self-Evolving Embodied Agents](https://arxiv.org/abs/2506.21669)
**Authors:** Wanxin Tian, Shijie Zhang, Kevin Zhang, Xiaowei Chi, Chunkai Fan, Junyu Lu, Yulin Luo, Qiang Zhou, Yiming Zhao, Ning Liu, Siyu Lin, Zhiyuan Qin, Xiaozhu Ju, Shanghang Zhang, Jian Tang
**TL;DR:** Proposes SEEA-R1, the first reinforcement fine-tuning (RFT) framework designed to enable self-evolving capabilities in embodied agents operating in multi-modal, long-horizon real-world settings. Addresses two core obstacles in embodied RFT: absence of accessible intermediate rewards in multi-step tasks, and over-reliance on hand-crafted reward functions that limit generalization. A tree-structured exploration strategy generates diverse rollouts and richer learning signals, enabling the agent to autonomously improve without environment-specific reward engineering.
**Tags:** `embodied-ai`, `reinforcement-learning`, `self-evolving`, `robotics`, `foundation-models`
