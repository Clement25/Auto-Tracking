# Top Papers — AI Agents & Embodied AI (2026-06-19)

## [AgentAtlas: Beyond Outcome Leaderboards for LLM Agents](https://arxiv.org/abs/2605.20530)
**Authors:** (AgentAtlas team, arXiv:2605.20530)
**TL;DR:** AgentAtlas argues that a single accuracy score is insufficient for evaluating deployable LLM agents and introduces a six-state control-decision taxonomy (Act / Ask / Refuse / Stop / Confirm / Recover) plus a nine-category trajectory-failure taxonomy. It also provides a benchmark-coverage audit mapping 15 agent benchmarks against six behavioral categories, enabling much richer diagnostic comparison.
**Tags:** `agents`, `evaluation`, `benchmarks`, `multi-agent`, `LLM`

---

## [LAP: Language-Action Pre-Training Enables Zero-shot Cross-Embodiment Transfer](https://arxiv.org/abs/2602.10556)
**Authors:** Lihan Zha, Asher J. Hancock, Mingtong Zhang, Tenny Yin, Yixuan Huang, Dhruv Shah, Allen Z. Ren, Anirudha Majumdar
**TL;DR:** LAP represents low-level robot actions directly in natural language, aligning action supervision with pre-trained vision-language model distributions. LAP-3B achieves the first substantial zero-shot transfer to previously unseen robot embodiments—over 50% average success across novel robots—roughly 2× better than prior VLAs, all without per-embodiment fine-tuning.
**Tags:** `embodied-ai`, `robotics`, `vision-language-action`, `cross-embodiment`, `zero-shot`

## [Multi-agent Embodied AI: Advances and Future Directions](https://arxiv.org/abs/2505.05108)
**Authors:** (Survey team, arXiv:2505.05108)
**TL;DR:** A comprehensive survey of multi-agent embodied AI systems covering recent advances in foundation models (including GR00T N1 for humanoids), multi-robot coordination, and shared perception. The paper maps out open research directions in communication, decentralized planning, and scalable sim-to-real pipelines.

## [Memory for Autonomous LLM Agents: Mechanisms, Evaluation, and Emerging Frontiers](https://arxiv.org/abs/2603.07670)
**Authors:** (arXiv:2603.07670)
**TL;DR:** This survey systematically reviews memory architectures for autonomous LLM-based agents—covering in-context, external, parametric, and episodic memory—along with evaluation protocols spanning 2022–2026. It identifies key gaps in long-horizon task performance and proposes a unified evaluation framework for memory-augmented agents.
**Tags:** `agents`, `memory`, `LLM`, `survey`, `long-horizon`

## [Self-evolving Embodied AI](https://arxiv.org/abs/2602.04411)
**Authors:** (arXiv:2602.04411)
**TL;DR:** This paper proposes self-evolving frameworks for embodied agents that iteratively refine their own policies through interaction, leveraging vision-language-action models and robotics transformers. The approach allows robots to progressively improve task performance without requiring large amounts of annotated demonstrations.
**Tags:** `embodied-ai`, `self-evolving`, `vision-language-action`, `robotics`, `continual-learning`

## [Spatial-Agent: Agentic Geo-spatial Reasoning with Scientific Core Concepts](https://arxiv.org/abs/2601.16965)
**Authors:** Riyang Bao, Cheng Yang, Dazhou Yu, Zhexiang Tang, Gengchen Mai, Liang Zhao
**TL;DR:** Spatial-Agent formalizes geo-analytical question answering by extracting foundational spatial concepts (Object, Field, Event, Network) and composing them into executable GeoFlow Graphs via template-based generation. It significantly outperforms baselines like ReAct and Reflexion on MapEval-API and MapQA benchmarks while producing interpretable geospatial workflows.
**Tags:** `agents`, `spatial-intelligence`, `geo-spatial`, `reasoning`, `tool-use`

---

## [ScaleSWE: Scaling Coding Agents via GitHub Data Mining](https://github.com/AweAI-Team/ScaleSWE)
**Authors:** (AweAI Team)
**TL;DR:** ScaleSWE mines millions of real GitHub pull requests to synthesize large-scale training data for coding agents, achieving 64% resolve rate on SWE-bench Verified with a fine-tuned Qwen-30B model. The released dataset includes 20,000 SWE task instances and 71k distillation trajectories, currently the largest open-source real-executable SWE dataset.
**Tags:** `agents`, `coding`, `SWE-bench`, `data-synthesis`, `LLM`

## [Phoenix: Safe GitHub Issue Resolution via Multi-Agent LLMs](https://arxiv.org/list/cs.MA/recent)
**Authors:** (IROS 2026 accepted paper)
**TL;DR:** Phoenix introduces a multi-agent LLM framework that safely resolves real GitHub issues by decomposing the task across specialized agents (planning, editing, testing, verification) with explicit rollback mechanisms. Accepted at IROS 2026, it demonstrates practical agentic software engineering with safety guarantees.
**Tags:** `agents`, `multi-agent`, `software-engineering`, `safety`, `LLM`

## [AgentHallu: Benchmarking Hallucination Detection in Multi-Step Agent Trajectories](https://arxiv.org/list/cs.AI/recent)
**Authors:** (arXiv, 2026)
**TL;DR:** AgentHallu provides a 693-trajectory benchmark spanning 7 agent frameworks and 5 domains, specifically designed to test whether models can identify which step in a multi-step execution caused a hallucination. This fine-grained, step-level evaluation enables more precise debugging and mitigation strategies for agentic LLM pipelines.
**Tags:** `agents`, `hallucination`, `evaluation`, `benchmarks`, `LLM`
